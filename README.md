<h2 align="center">How I Work</h2>
<p align="center">
  <samp>
    My development process is built around a few core principles:
  </samp>
</p>
<table align="center" width="90%">
  <tr>
    <td width="50%" valign="top">
      <h4>Architecture First</h4>
      <sub>
        I think in systems before lines of code. Every project starts with a clear mental model of data flow, state management, and deployment topology. This prevents costly rewrites later.
      </sub>
      <br/><br/>
      <h4>Ship Early, Iterate Fast</h4>
      <sub>
        I break work into vertical slices that deliver value incrementally. Working demos over perfect abstractions. Feedback loops drive refinement, not upfront speculation.
      </sub>
    </td>
    <td width="50%" valign="top">
      <h4>Performance by Design</h4>
      <sub>
        I optimize at the architecture level — efficient queries, proper indexing, lazy loading, CDN distribution, and caching strategies. Not afterthought patches.
      </sub>
      <br/><br/>
      <h4>Own the Full Stack</h4>
      <sub>
        From database schema to pixel-perfect UI, I understand how every layer interacts. This means fewer handoff gaps, faster debugging, and more coherent products.
      </sub>
    </td>
  </tr>
</table>
